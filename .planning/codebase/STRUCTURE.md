## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router routes and layouts
- Contains: Page components (`page.tsx`), layouts (`layout.tsx`), API routes
- Key files: `src/app/layout.tsx` (root layout), `src/app/(protected)/layout.tsx` (protected routes), `src/app/api/[[...slugs]]/route.ts` (API router)

**`src/components/`:**
- Purpose: Reusable React components organized by feature
- Contains: Feature-specific components, UI components (shadcn)
- Key files: `src/components/ui/` (shadcn components), `src/components/common/` (shared layout components)

**`src/server/`:**
- Purpose: Backend logic including API, database, auth, and MCP server
- Contains: Elysia routes, Drizzle schema, Better Auth config, MCP tools
- Key files: `src/server/elysia/index.ts` (API router), `src/server/db/schema.ts` (database schema), `src/server/better-auth/config.ts` (auth config)

**`src/hooks/`:**
- Purpose: Custom React hooks for data fetching and state management
- Contains: TanStack Query hooks for API calls, utility hooks
- Key files: `src/hooks/use-scholarships.ts`, `src/hooks/use-auth.ts`

**`src/lib/`:**
- Purpose: Client-side utilities and configurations
- Contains: Auth client, Eden client, S3 client, helper functions
- Key files: `src/lib/auth-client.ts`, `src/lib/eden.ts`, `src/lib/s3.ts`

**`drizzle/`:**
- Purpose: Database migration files generated by Drizzle Kit
- Contains: `migration.sql` and `snapshot.json` for each migration
- Committed: Yes

**`test/`:**
- Purpose: Integration and end-to-end tests
- Contains: API integration tests, MCP tests
- Key files: `test/specs/study-plans.spec.ts`, `test/integration/mcp/`

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root layout with providers, Navbar, Toaster
- `src/app/page.tsx`: Landing page
- `src/app/api/[[...slugs]]/route.ts`: Elysia API router export
- `src/app/api/mcp/[transport]/route.ts`: MCP server handler

**Configuration:**
- `src/env.js`: Environment variables schema with Zod validation
- `src/server/db/index.ts`: Database connection (Drizzle + postgres-js)
- `src/server/better-auth/config.ts`: Better Auth configuration
- `next.config.ts`: Next.js configuration
- `drizzle.config.ts`: Drizzle Kit configuration

**Core Logic:**
- `src/server/elysia/index.ts`: Main Elysia API instance with all routes
- `src/server/db/schema.ts`: Central schema file importing all tables
- `src/server/db/relations.ts`: Drizzle relations for nested queries
- `src/lib/eden.ts`: Eden Treaty client for type-safe API calls

**Testing:**
- `test/`: Integration tests and E2E specs
- `test/specs/`: Playwright or similar E2E test specs
- `test/integration/`: API integration tests

## Naming Conventions

**Files:**
- kebab-case for all files: `scholarship-card.tsx`, `use-scholarships.ts`, `api-keys.ts`
- Route groups in parentheses: `(protected)/`, `(verification)/`

**Directories:**
- kebab-case for feature directories: `course-explorer/`, `study-planner/`, `resume-builder/`
- `ui/` for shadcn components

**Database Tables:**
- snake_case for table names: `scholarship`, `scholarship_round`, `user_profile`
- PascalCase for Drizzle table variables: `scholarships`, `scholarshipRounds`

**API Routes:**
- kebab-case in URLs: `/api/scholarships`, `/api/colleges`, `/api/study-plans`
- Multi-word: dash-separated

**Components:**
- PascalCase for component names: `ScholarshipCard`, `ProtectedRoute`, `Navbar`
- kebab-case for files: `scholarship-card.tsx`, `protected-route.tsx`

**Hooks:**
- camelCase with `use` prefix: `useScholarships`, `useAuth`, `useInfiniteScroll`

## Where to Add New Code

**New Feature (API + Frontend):**
- Primary code:
  - API routes: `src/server/elysia/routes/[feature].ts`
  - Database schema: `src/server/db/schema/[feature].ts` (if new tables)
  - Frontend components: `src/components/[feature]/`
  - Pages: `src/app/[route-group]/[feature]/page.tsx`
- Tests: `test/integration/[feature].test.ts` or `test/specs/[feature].spec.ts`

**New API Endpoint:**
- Implementation: `src/server/elysia/routes/[domain].ts`
- Register in: `src/server/elysia/index.ts` (add `.use(featureRoutes)`)
- Client hook: `src/hooks/use-[domain].ts`
- Types: Export from `src/app/api/[[...slugs]]/route.ts` as part of `App` type

**New Component/Module:**
- UI components: `src/components/[feature]/[component-name].tsx`
- Page routes: `src/app/[route-group]/[route]/page.tsx`
- Layout: `src/app/[route-group]/layout.tsx` (if needed)

**Database Schema Changes:**
- Schema definition: `src/server/db/schema/[domain].ts`
- Export from: `src/server/db/schema.ts` (add `export * from "./schema/[domain]"`)
- Relations: `src/server/db/relations.ts` (add relations if needed)
- Migration: Run `bun run db:generate` then `bun run db:migrate`

**Utilities:**
- Shared helpers: `src/lib/utils.ts`
- Feature-specific utilities: `src/lib/[feature].ts` (create new file)

**MCP Tools:**
- Tool implementation: `src/server/mcp/tools/[domain].ts`
- Register in: `src/server/mcp/server.ts` (call `register[Domain]Tools(server)`)

## Special Directories

**`src/app/(protected)/`:**
- Purpose: Route group for pages requiring authentication
- Generated: No
- Committed: Yes
- Pattern: All pages wrapped with `ProtectedLayout` component

**`src/app/(verification)/`:**
- Purpose: Route group for email verification, 2FA flows
- Generated: No
- Committed: Yes

**`src/components/ui/`:**
- Purpose: Shadcn UI components (auto-generated via shadcn CLI)
- Generated: Yes (via `bunx shadcn@latest add [component]`)
- Committed: Yes
- Do not edit directly - customize by creating wrapper components

**`drizzle/`:**
- Purpose: Database migrations
- Generated: Yes (via `bun run db:generate`)
- Committed: Yes
- Never edit manually - regenerate when schema changes

**`src/assets/icons/`:**
- Purpose: Custom icon components (from Lucide or custom SVGs)
- Generated: No
- Committed: Yes

**`public/`:**
- Purpose: Static assets served at root URL
- Generated: No (except during build)
- Committed: Yes
